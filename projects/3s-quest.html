3S Quest: Tidy Up The Workplace

VR Workplace Simulation

Project Overview

A virtual reality simulation game where players must follow the 3S principles: Sorting, Set in Order, and Shine. Players clean, organize, and manage workplace tasks within a time limit to achieve high scores — testing speed, accuracy, and prioritization through realistic VR interactions.

Documentation

Key Features

  • Immersive VR Workplace Simulation
  • Interactive Object Handling
  • Task-based Progression System
  • Dynamic Scoring and Grading System
  • Custom Task Mechanics for Each Map
  • Integrated Timer & Result System

Technical Details

  • Role: Game Programmer & Game Designer
  • Team Size: 4 People
  • Time Frame: 2 Months
  • Engine / Tools: Unity
  • Game Platform: Meta Quest 2
  • Released: May, 2025

My Contributions / What I Have Learned

In this VR 5S simulation project, my primary responsibility was developing and implementing the entire gameplay system for Level 2 in both Map 1 and Map 2. I designed and programmed the core mechanics for Sorting, Set in Order, and Shine, adjusting each task’s rules and scoring logic based on real 5S principles. I handled task progression tracking, scoring calculations, penalty systems for incorrect task order, and special conditions like SortBias item detection. Additionally, I built the result assessment system that calculates scores, grades, and final time, complete with a result scene UI. I also integrated sound effects for every interaction, created start/stop gameplay triggers, and managed audio and background music looping systems. Beyond coding, I contributed by preparing user guides, feature documentation, and assisting in balancing the scoring system to ensure fair and challenging gameplay. My work helped shape Level 2 into a complete, functional, and immersive VR training environment.

Throughout this VR 5S game project, I gained valuable experience in managing multi-task interaction systems within a VR environment. I learned how to structure and implement clean, modular scripts for complex task flows like sorting, object snapping, and task-based scoring systems. This project taught me the importance of prioritizing task order in gameplay logic, handling dynamic scoring adjustments, and designing fair assessment systems based on player performance. I also developed skills in integrating sound effects and UI result systems to enhance immersion and player feedback. Additionally, I learned to collaborate with other developers while maintaining separate, non-conflicting scripts for multi-scene projects, and how to document features and user guides clearly for both players and team members. This project strengthened my confidence in Unity VR development, C# scripting, and overall game mechanic balancing.
